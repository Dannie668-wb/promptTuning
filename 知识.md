  - APE (Auto Prompt Engineer) — 自动生成并筛选候选提示词

  3. 评估 + 迭代框架

  ┌─────────────────────────────────────────────┐
  │           评估驱动优化循环                    │

      └──────────── 修改提示词 ◀─────────────┘

  - PromptFoo — 开源，支持对比测试多个提示词版本
  - LangSmith — LangChain 生态，追踪 + 评估 + 可视化
  - Promptflow (Azure) — 企业级，可视化流程编排
  - Braintrust — 专注 LLM 评估，A/B 测试提示词

  4. 专项工具

  - Anthropic 提示词生成器 — 输入任务描述，自动生成结构化系统提示词（Console 内置）
  - Prompt Perfect — 浏览器插件，实时润色你的输入
  - PromptBase — 买卖高质量提示词的市场



## 实践

我们收集和设计的数据式JsonSchema，那怎么跟a2ui进行关联                                  

● 这是整个系统最核心的问题。答案是一个 Component Registry（组件注册表），它是 JSON Schema 和 a2ui 之间的桥梁。

  ---
  绑定机制全景

  ┌─────────────────────────────────────────────────────────────┐
  │                   Component Registry                         │
  │              (JSON Schema ↔ a2ui 的桥梁)                    │
  └─────────────────────────────────────────────────────────────┘

    JSON Schema 里的 "type": "Input"
              │
              ▼
    Registry 查找 "Input" 条目
              │
       ┌──────┴──────┐
       ▼             ▼
    实际组件        Schema 定义
    (渲染用)       (配置面板用)
       │             │
       ▼             ▼
    <AdmInput/>   props: [label, placeholder, required...]
                  style: [marginBottom, padding...]
                  events: [onChange → setField]

  ---
  每个组件的双重定义

  // src/components/Input/index.ts

  export const InputDefinition: ComponentDefinition = {

    // 1. 元信息
    type: "Input",
    label: "输入框",
    icon: "input-icon",

    // 2. 实际渲染的 React 组件（来自 ant-design-mobile）
    component: AntdInput,

    // 3. 默认 Schema（拖入画布时的初始值）
    defaultSchema: {
      type: "Input",
      props: { label: "标签", placeholder: "请输入" },
      style: {},
      events: {}
    },

    // 4. 配置面板的元数据（告诉配置面板渲染哪些控件）
    propsConfig: [
      { key: "label",       type: "string",  label: "标签文本" },
      { key: "placeholder", type: "string",  label: "占位符"   },
      { key: "required",    type: "boolean", label: "必填"     },
      { key: "disabled",    type: "boolean", label: "禁用"     },
    ],
    styleConfig: [
      { key: "marginBottom", type: "string", label: "下边距" },
      { key: "width",        type: "string", label: "宽度"   },
    ],
    eventsConfig: [
      { key: "onChange", label: "值变化时", allowedActions: ["setField"] },
      { key: "onBlur",   label: "失焦时",   allowedActions: ["setField", "showToast"] },
    ]
  }

  ---
  三者如何联动

  ┌──────────┐   拖拽   ┌──────────────┐  选中组件  ┌──────────────┐
  │ 组件面板  │────────▶│   预览区      │──────────▶│  配置面板     │
  │          │         │              │           │              │
  │ [Input]  │         │ <Input/>     │           │ label: [  ]  │
  │ [Form]   │         │ (真实交互)    │           │ required:[✓] │
  └──────────┘         └──────────────┘           └──────┬───────┘
        │                      ▲                          │
        │                      │                          │ 编辑
        ▼                      │                          ▼
    defaultSchema          Renderer               更新 JSON Schema
    注入 Schema            Schema→组件              (状态管理)


## 整体流程
  手动辅助接入点

  ┌─────────────────────────────────────────────────────────────────┐
  │              完整优化循环（含手动辅助）                           │
  └─────────────────────────────────────────────────────────────────┘

    初始3个示例
         │
         ▼
    ┌─────────┐
    │  DSPy   │──── 优化 ────▶ prompt_v1.txt ──▶ 生产环境
    │  优化   │
    └─────────┘
                                    │
                              用户真实请求
                                    │
                                    ▼
                            ┌───────────────┐
                            │  LangSmith    │
                            │  自动评估     │──▶ schema合法? 组件合规?
                            │  + 打标记     │
                            └───────┬───────┘
                                    │
                            ┌───────▼───────┐
                            │  人工标注队列  │  ← 手动辅助入口 1
                            │  (Annotation  │
                            │   Queue)      │
                            └───────┬───────┘
                                    │
                ┌───────────────────┼───────────────────┐
                ▼                   ▼                   ▼
            评为"好"           评为"差"           人工修正后保存
                │                   │                   │  ← 手动辅助入口 2
                └───────────────────┴───────────────────┘
                                    │
                            扩充训练集 (examples/)
                                    │
                                    ▼
                            重跑 DSPy 优化
                                    │
                                    ▼
                            prompt_v2.txt ──▶ 覆盖生产

  ---
  手动辅助的两个入口

  入口 1：LangSmith Annotation Queue（评分）
  入口 2：人工修正后存为新训练样本
