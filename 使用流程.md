# a2ui AI Page Generator — 使用流程

## 整体架构概览

```
用户描述
   │
   ▼
FastAPI /generate
   │  加载 prompts/current_prompt.txt
   │  填入 {description}
   ▼
DeepSeek API  ──── LangSmith 全链路追踪
   │
   ▼
PageBundle JSON
   │
   ├── 自动入审核队列（AUTO_ANNOTATION_QUEUE=true 时）
   │        │
   │        ▼ 人工标注
   │   LangSmith Annotation Queue
   │        │ 分数回流
   │        ▼
   └── DSPy 优化 → 写入 prompts/
```

---

## 一、手动辅助（Playground）

### 目的

在不重新训练的前提下，快速试验和微调提示词，并把好的例子保存为训练数据。

### 操作步骤

**1. 测试提示词**

```http
POST /playground/test
{
  "description": "商品列表页，支持按价格筛选",
  "custom_prompt": "（留空则使用当前版本）",
  "version": "20260610_001821",   // 指定历史版本，可选
  "temperature": 0.2,
  "n_samples": 3                  // 同一描述生成多个结果，用于对比
}
```

返回 `bundles` 数组，可逐一检查生成质量。

**2. 保存人工修正的样本**

发现生成结果有问题时，在前端直接修正后调用：

```http
POST /playground/save-example
{
  "description": "商品列表页，支持按价格筛选",
  "corrected_bundle": { ... },   // 手工修正后的正确 JSON
  "run_id": ""                   // 可选，关联 LangSmith run
}
```

修正后的样本会追加到 `frontend/src/examples/training_data.json`，供下次 DSPy 优化使用。

**3. 手动编辑提示词并保存为新版本**

直接在 Playground UI 中修改提示词文本，然后：

```http
POST /playground/save-version
{
  "prompt_text": "（修改后的完整提示词文本）"
}
```

- 新版本写入 `prompts/{时间戳}_prompt_template.txt`
- 同时覆盖 `prompts/current_prompt.txt` 立即生效
- 返回 `version` 字段（时间戳格式：`20260610_153022`）

**4. 查看历史版本列表**

```http
GET /playground/versions
```

返回所有版本，按修改时间倒序排列。

## 二、自动优化（DSPy）

### 目的

用 DSPy `BootstrapFewShot` 从训练数据中自动筛选最优 few-shot 示例，生成新提示词。

### 前置条件

- `frontend/src/examples/training_data.json` 中有至少 3 条标注数据
- `DEEPSEEK_API_KEY` 已配置

### 执行优化

在 `backend/` 目录下：

```bash
python -m ai.dspy.optimize_and_export
```

**运行过程：**

1. 加载 `training_data.json`，自动校验每条样本的 PageBundle 合规性
2. DSPy 用 `BootstrapFewShot` 在训练集上试跑，每条样本按 `schema_metric` 评分
3. 自动选出最优的 few-shot 示例组合
4. 调用一次 DeepSeek（触发词 `__export_trigger__`）把优化后的完整 prompt 固化为文本
5. 导出文件

### 评分标准（schema_metric）

| 项目 | 分值 |
|------|------|
| 输出是合法 JSON | +0.2 |
| PageBundle 完全符合规范（无错误） | +0.6 |
| 规范错误每个 | -0.1（最低 0） |
| LangSmith 人工标注为正面 | +0.2 |
| 总分上限 | 1.0 |

## 三、评估驱动的执行（LangSmith）

### 全链路追踪

所有 `/generate` 和 `/playground/test` 调用都通过 `@traceable` 自动上报到 LangSmith，记录：
- 输入（description）
- 输出（PageBundle）
- 延迟、token 用量

LangSmith Project 名：`a2ui-page-generator`（在 `.env` 中配置）

### 人工标注队列

**自动入队（生产模式）：**

在 `.env` 中设置：

```
AUTO_ANNOTATION_QUEUE=true
```

每次 `/generate` 调用后，run 会自动进入 LangSmith 队列 **"a2ui页面生成质量评估"**。

**手动入队：**

```python
from ai.langsmith.annotation import submit_for_review
submit_for_review(run_id)
```

**在 LangSmith UI 中标注：**

1. 打开 LangSmith → Annotation Queues → "a2ui页面生成质量评估"
2. 逐条审查生成结果，打分（正分 = 好，0 或负分 = 差）
3. 分数自动保存，下次运行 DSPy 时通过 `_fetch_human_score` 回流，影响 `schema_metric`

### 评估驱动的优化循环

```
生产调用 → LangSmith 追踪 → 人工标注
                                  │
                         ┌────────▼────────┐
                         │ schema_metric   │
                         │ 含人工分 +0.2   │
                         └────────┬────────┘
                                  │
                         DSPy 优化筛选
                                  │
                         新版提示词写入
                         prompts/current_prompt.txt
                                  │
                         下次生产调用自动使用
```

## 四、生成的提示词位置

所有提示词文件统一存放在项目根目录的 `prompts/` 文件夹：

```
prompts/
├── current_prompt.txt               ← 当前生产使用的版本（始终最新）
├── 20260610_001821_prompt_template.txt  ← DSPy 或手动保存的历史版本
├── 20260610_001821_dspy_program.json    ← DSPy 程序状态（可用 dspy.Predict.load() 恢复）
└── ...
```

**文件命名规则：**

- 时间戳格式：`YYYYMMDD_HHMMSS`（UTC，DSPy 导出） / 本地时间（手动保存）
- `*_prompt_template.txt`：人类可读的完整提示词文本（含 few-shot 示例）
- `*_dspy_program.json`：DSPy 内部状态，仅 DSPy 优化时产生

**读取当前提示词：**

```python
from api.prompt_manager import load_prompt_version

current = load_prompt_version()          # 读取 current_prompt.txt
v = load_prompt_version("20260610_001821")  # 读取特定版本
```

**提示词中的占位符：**

- `{description}` — 运行时替换为用户输入的页面描述

## 五、手动优化 vs 自动优化的对比

### 并排比较方法

**步骤 1：确认要对比的两个版本**

```http
GET /playground/versions
# 返回示例：
# [
#   {"version": "20260610_153022", "path": "..."},  ← 手动版
#   {"version": "20260610_001821", "path": "..."}   ← DSPy 版
# ]
```

**步骤 2：用同一组描述分别测试**

```http
POST /playground/test
{"description": "用户信息编辑表单", "version": "20260610_001821", "n_samples": 1}

POST /playground/test
{"description": "用户信息编辑表单", "version": "20260610_153022", "n_samples": 1}
```

两次请求的输出 bundle 通过前端或脚本并排展示，对比：
- JSON 结构是否合规
- 组件选用是否合理
- initialData 绑定路径是否正确

**步骤 3：用 DSPy metric 脚本批量评分（可选）**

```python
from ai.dspy.optimize_and_export import _validate_page_bundle
import json

bundle = json.loads(result["bundles"][0])
is_valid, errors = _validate_page_bundle(bundle)
print(f"合规: {is_valid}, 错误: {errors}")
```

**步骤 4：决策**

| 情况 | 建议 |
|------|------|
| DSPy 版合规率更高但语义差 | 补充更多训练样本再优化 |
| 手动版语义好但偶发格式错误 | 手动版为基础 + 修正格式规则后保存 |
| 两版相近 | 保留 DSPy 版（可复现） |

**将胜出版本设为当前版本：**

```http
POST /playground/save-version
{"prompt_text": "（从 prompts/{version}_prompt_template.txt 读取的文本）"}
```

## 六、快速入门检查清单

```
□ 配置 .env（DEEPSEEK_API_KEY、LANGSMITH_API_KEY、LANGSMITH_PROJECT）
□ 启动后端：cd backend && uvicorn main:app --reload
□ 首次使用：准备 3+ 条训练数据，运行 python -m ai.dspy.optimize_and_export
□ 检查 prompts/current_prompt.txt 是否生成
□ 调用 POST /generate {"description": "..."} 验证生成效果
□ 在 LangSmith 中标注结果质量
□ 积累足够标注后，重新运行 DSPy 优化，对比新旧版本
```
